Ruby: Arrays vs Hashes

Arrays

According to Ruby-Doc, "Arrays are ordered, integer-indexed collections of any object." Or put simply, an ordered list of things. They are used to store collections of data. Each object in an array can be accessed using a unique key. The first element in the array can be accessed using index 0 of the array.

Example: favorite_fruits_array = ["orange", "apple", "mango", "strawberry", "kiwi"] To access "mango", you would do favorite_fruits_array[2].

Reference: Ruby-Doc

Hashes

According to Ruby Doc, "A Hash is a collection of key-value pairs. It is similar to an Array, except that indexing is done via arbitrary keys of any object type, not an integer index." Key-value what? You could picture it like this: Kevin went to Syracuse, John went to Harvard, and Mike went to UCLA. The names of the people would be the key and the schools would be the value. Hashes point an object to another object.

Example: school_backgrounds = { 'Kevin' => 'Syracuse', 'John' => 'Harvard', 'Mike' => 'UCLA' } To access what school John went to, you would do: school_backgrounds['John']. The result would be => "Harvard".

Reference: Ruby-Doc

When to use them

It really depends on the scale of the data you are storing as well as what is being stored. As you saw above, arrays are accessed using indexes. When trying to find a very specific element in the array, it could get tedious and difficult to find exactly what you want. Hashes are easy to access if you are looking for something specific, because you can point straight to the key to access the value. However, hashes are not ordered data. It becomes quite complex when trying to get the data out of a hash back in an ordered way. This is where arrays become more useful than hashes. If the ordering is important to you, it is safe to use arrays over hashes.
