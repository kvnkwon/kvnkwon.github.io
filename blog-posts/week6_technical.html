Class Methods

Last week, I talked about instance methods and how they exist at the object level. This week, we'll be taking a look into class methods that exist at the class level.

If we look at this class here, we can see that the class has an instance method.

Lets now take a look at an example of a class method. Both classes from the examples have the same function and output. Can you notice what changed?

You can see that there is no initialize method for the second example because it uses the class method, self.display, which accepts a title and artist for its parameters. The .self in this case is just telling the class, that display is a class method. The first example required us to call a new instance of the class in order to use the instance method, but the second example allowed us to call the class method, display, directly on the class without calling a new instance of it. Sweet right?

Class methods are global methods, which just means you can access and call them from anywhere without creating an object instance. Instance methods are part of object-oriented programming, whereas class methods are part of procedural programming. Many developers avoid using class methods, because they favor the object oriented approach. Class methods may be handy and simple in solving a solution, but ultimately theres going to be a better approach to it through Object Oriented Programming.
