What is Test Driven Development(TDD)?

Test driven development involves the developer to first write some code that will fail. The developer will write just enough code so that the direction of the code is clear, but it must fail. The next step is to update that code so that the tests will pass. If the tests pass, the process repeats itself, and the developer writes more code that will indeed fail. After all the code has been written and passed, it is time to refactor by making the code as non-redundant as possible.

The above graphic does a good job in concisely showing how TDD works. The quote, "red, green, refactor," is pretty much the guiding three words for TDD. Failure is red, passing is green, and when it's green, you must refactor.
