Instance Variables and Methods

Instance Variables

Have you worked with instance variables before? No? You can spot an instance variable by the signature "@" sign in front of them, recognize them now? It's okay if you don't, but you will get used to them while you code in Ruby. "They differ from local variables in that they don't exist within any particular scope. Instead, a similar variable table is stored for each instance of a class. Instance variables live within a class instance, so as long as that instance stays alive, so will the instance variables (source)." Instance variables are awesome in that you can use them in any method of that specific class. Local variables are restricted and can only be used in their specific method.

Instance Methods

The previous example above actually had an instance method, "initialize." A key point about instance methods is that they only work with an instance, which requires you to create a new instance of the class to use them. What does that mean? Lets take a look.
